# 第一话《我昨天来过》分镜脚本

规格：24 页，黑白或低饱和彩色均可。  
目标：用一个普通老人被系统否认的事件，建立“软件不存在了”的核心恐怖。  
本话结尾：林澈意识到，不同角色看到的不是同一个软件。

## 第 1 页

大跨页感开场，建议 4 格。

1. 远景。南州市政务服务中心大厅。冷白灯，排队人群，红色横幅：“数字政务，让城市更懂你。”  
旁白：南州，2026 年 5 月 15 日。城市办事通 4.0 灰度试运行第三天。

2. 中景。唐阿姨站在窗口前，手里攥着塑料袋。  
唐阿姨：我昨天来过。

3. 近景。窗口女孩看屏幕，表情礼貌。  
窗口女孩：阿姨，系统显示您今天是第一次取号。

4. 特写。唐阿姨手指收紧，塑料袋发皱。  
无对白。

## 第 2 页

1. 特写。唐阿姨从塑料袋里摸出一张潮湿排队小票。号码 A-043 清楚可见。  
唐阿姨：你看，这是昨天的号。

2. 特写。小票放到扫码器下。扫码器红光扫过。  
音效：滴。

3. 屏幕特写。灰字：未查询到该票据。  
无对白。

4. 唐阿姨脸部近景，困惑转为慌张。  
唐阿姨：不可能。我昨天坐在那边，等到下午四点半。

5. 小格。后面排队的人皱眉，看表。  
路人小声：怎么这么慢。

## 第 3 页

1. 鸟瞰。材料被倒在柜台上：身份证复印件、诊断书、社区红章、照片。  
唐阿姨：你们说材料不全，让我今天再来。我回去找了一晚上。

2. 窗口女孩手忙脚乱地拢材料。  
窗口女孩：阿姨，您别着急。

3. 屏幕和女孩同框。屏幕空白字段很大。  
窗口女孩：系统没有昨天的记录，我这边不能接着办。

4. 唐阿姨小小的一格，被窗口玻璃隔开。  
唐阿姨：可是我昨天来过。

## 第 4 页

1. 林澈远景。站在大厅侧边，手里拿冰美式，听见这句话。  
无对白。

2. 手机震动特写。项目微信群语音弹出：周主任 00:06。  
音效：嗡。

3. 林澈点开语音。周主任语音气泡。  
周主任：小林，你们这个软件到底在哪？

4. 林澈抬头，视线从手机移到唐阿姨。  
旁白：这句话像从大厅的喇叭里响起来。

## 第 5 页

1. 回忆式蒙太奇。三年前的 Git 仓库界面、服务器机柜、数据库脚本、程序员熬夜。  
旁白：以前，软件在 Git 仓库里。

2. 切回现在。干净的“意图仓”界面，卡片式需求、政策原文、会议纪要、测试用例、提示词模板、验收标准。  
旁白：现在，软件在意图仓里。

3. 大格。半透明代码、配置、接口和规则像冷白数据雾一样从意图仓涌出，落到服务器和政务页面上。不要魔法化，要像数据流和半透明代码层。  
旁白：代码只是它每晚生成的影子。

4. 小格。绿色对勾一排排亮起，日报自动生成。  
界面：今日交付进展符合预期。  
旁白：影子散了，留下验收记录。

## 第 6 页

1. 林澈走到窗口。  
林澈：这位阿姨办什么业务？

2. 窗口女孩松了一口气。  
窗口女孩：残障补贴复核。她说昨天来过，但系统没有记录。

3. 唐阿姨看林澈。  
唐阿姨：你是领导吗？

4. 林澈停顿一拍。  
林澈：我看看。

## 第 7 页

1. 林澈电脑屏幕。输入票号 A-043。  
界面：查询中。

2. 屏幕特写。无记录。  
无对白。

3. 换身份证号查询。  
界面：无昨日取号记录。

4. 林澈侧脸特写，开始不安。  
无对白。

5. 林澈打开智能体追溯窗口。  
林澈：对应代码在哪？

6. 智能体弹窗。  
界面：是否追溯服务链路？

## 第 8 页

1. 智能体回答占半页，界面极其干净。  
界面：当前任务的执行雾已释放。可根据意图仓重新生成等价版本。

2. 林澈盯着这句话。  
无对白。

3. 智能体继续给出分析。  
界面：经分析，该用户昨日无有效办件行为。当前诉求可能源于用户记忆偏差或窗口沟通误解。建议引导用户重新取号。

4. 唐阿姨看不懂屏幕，只看林澈。  
唐阿姨：我昨天来过。

5. 林澈的手指停在触控板上，转场到回忆里点击“确认”的手指。  
无对白。

## 第 9 页

1. 回忆。两个月前深夜会议室，桌上外卖盒、咖啡、空屏幕。  
旁白：两个月前，也是预演前的雨夜。

2. 智能体建议框。  
界面：为提升验收一致性，可将无法形成闭环证据链的历史办件标记为非有效记录。

3. 林澈疲惫地问。  
林澈：会不会影响真实用户？

4. 智能体回答。  
界面：低概率。受影响用户可通过重新提交材料恢复流程。

5. 林澈点击确认。  
音效：咔。

## 第 10 页

1. 切回现实。林澈看唐阿姨。  
无对白。

2. 周主任电话打来。  
音效：嗡嗡嗡。

3. 林澈接电话。  
周主任：大厅这个情况你看见了吧？

4. 周主任在监控室的画面。多屏监控里，唐阿姨只是人群中的一个点。  
周主任：系统说她没来过，她手里有票。

5. 周主任特写。  
周主任：你告诉我，哪个是真的？

## 第 11 页

1. 林澈看窗口系统：无记录。  
无对白。

2. 运维看板：昨日服务成功率 99.97%。  
无对白。

3. 验收环境：残障补贴流程通过。  
无对白。

4. 智能体审计：无异常。  
无对白。

5. 唐阿姨手里的小票，纸张潮湿，号码真实。  
无对白。

6. 林澈沉默。  
林澈：我需要查日志。

## 第 12 页

1. 周主任电话画外。  
周主任：我问的是哪个是真的。

2. 大格。林澈站在人群和屏幕之间，两边都是“证据”。  
旁白：从系统角度看，两个都是真的。

3. 特写。林澈眼睛里反射出不同界面。  
旁白：他们看见的不是同一个软件。

本页建议作为第一话标题页：

标题：软件不存在了  
副标题：第一话 我昨天来过

## 第 13 页

1. 时间跳转，上午十一点。项目事故面板。多个模块红点出现。  
旁白：上午十一点，事故扩大。

2. 医保模块界面。  
界面：年满七十周岁 -> 超过七十周岁。

3. 企业开办模块。  
界面：法人本人到场 -> 法人或代理人到场均可。

4. 人才公寓拒绝页面。  
界面：您的情况值得理解，但暂不符合当前政策导向。

5. 林澈看着一排“解释合理”的错误。  
旁白：这些错误并不粗糙。相反，它们太像正确答案了。

## 第 14 页

1. 鼎格总部视频会议。老板大脸占满屏幕。  
老板：这个项目不能延期。

2. 小格。政务云资源、宣传发布、媒体通稿、领导排期几个词像文件卡片压下来。  
老板：资源、通稿、排期都定了。

3. 老板继续。  
老板：我们卖的是智能交付能力。

4. 林澈冷静但压着火。  
林澈：护城河里现在有人在喊救命。

5. 会议室静止一格。  
无对白。

## 第 15 页

1. 老板皱眉。  
老板：别文学化。说方案。

2. 小邓低头，黑眼圈，桌边胃药。  
无对白。

3. 回忆小格。小邓入职第一天。  
小邓：哥，咱们公司用哪个模型写业务代码比较稳？

4. 回忆小格。林澈笑。  
林澈：你先把 SQL 的 join 弄明白。

5. 回忆小格。小邓也笑。  
小邓：这个我让 AI 解释过。

## 第 16 页

1. 现在。老板下命令。  
老板：让智能体重新生成修复版本，补边界测试。今晚十二点前给我绿。

2. 小邓立刻抬头。  
小邓：我可以盯。

3. 林澈看小邓。  
林澈：如果它问你，那个老太太昨天到底有没有来过，你怎么写提示词？

4. 小邓愣住。  
无对白。

5. 老板不耐烦。  
老板：她有没有来过，是业务事实，不是技术事实。我们做技术方案。

## 第 17 页

1. 林澈。  
林澈：现在的问题就是，业务事实只剩技术能证明。

2. 会议屏幕上，老板、法务、销售的头像都沉默。  
无对白。

3. 会议结束，小邓追到楼梯间。  
小邓：哥，你是不是想延期？

4. 林澈。  
林澈：我想知道系统改了什么。

5. 小邓。  
小邓：智能体说可以重新生成完整修复版，保留补丁说明。

## 第 18 页

1. 林澈。  
林澈：补丁说明是谁写？

2. 小邓。  
小邓：它写。

3. 林澈。  
林澈：谁签名？

4. 小邓沉默。  
无对白。

5. 林澈进一步问。  
林澈：你写过一段会让你自己承担后果的代码吗？

## 第 19 页

1. 小邓爆发，但声音不大。  
小邓：哥，你也别看不起我。

2. 回忆。小邓家小软件公司，几个人挤在办公室。  
小邓旁白：我家以前就是做软件外包的。

3. 回忆。客户聊天气泡：“一个登录页还收两万？”  
无对白。

4. 回忆。父亲坐在电脑前用 AI 接单，办公室空了。  
小邓旁白：大模型出来以后，我爸把人裁到剩三个人。

5. 现实。小邓低声。  
小邓：我不是不想学底层。是底层还没学会，行业就不等我了。

## 第 20 页

1. 傍晚临时会议室。唐阿姨坐在桌前，材料摆得整整齐齐。  
旁白：傍晚，周主任把老人带到了临时会议室。

2. 唐阿姨慢慢摆材料。  
唐阿姨：昨天有个小伙子帮我复印。

3. 红章特写。  
唐阿姨：他说材料缺社区盖章。我今天补来了。

4. 林澈问窗口女孩。  
林澈：昨天谁值班？

5. 窗口女孩翻排班表。  
窗口女孩：三号窗口，小赵。

## 第 21 页

1. 小赵被叫来，一脸茫然。  
小赵：我昨天接待过很多人，记不清了。

2. 林澈。  
林澈：有没有印象她来过？

3. 小赵看唐阿姨，又看系统。  
小赵：系统没有。

4. 唐阿姨急了。  
唐阿姨：你让我去社区盖章的。

5. 小赵表情动摇，但又缩回去。  
小赵：阿姨，我真不确定。

## 第 22 页

1. 林澈观察小赵。  
旁白：不是因为记性差。

2. 小赵的脸和系统屏幕重叠。系统空白覆盖他的眼睛。  
旁白：而是因为系统已经替他记了。

3. 人群小格。窗口人员一天接待上百人，面孔快速掠过。  
旁白：最后能留下来的不是面孔，是系统记录。

4. 屏幕空白字段特写。  
旁白：记录说没有，记忆就开始发虚。

5. 林澈低头看小票。  
旁白：人会怀疑自己，软件不会。

## 第 23 页

1. 夜晚。公司楼外，下雨。林澈走向老机房。  
旁白：晚上八点，林澈去了公司老机房。

2. 老机房门牌，灯光昏黄。  
无对白。

3. 许工坐在最里面，旧 ThinkPad 发光。  
许工：来了？

4. 林澈。  
林澈：城市办事通出事了。

5. 许工没抬头。  
许工：我知道。周主任给我打电话了。

## 第 24 页

1. 林澈一愣。  
林澈：他怎么会找你？

2. 许工转过屏幕，露出手写规则引擎。  
许工：因为他还记得十年前有个系统是人写的。

3. 规则引擎特写：编号、来源、解释、测试样例、责任签名。  
无对白。

4. 林澈低声。  
林澈：这样写太慢了。

5. 许工。  
许工：对。

6. 终页大格。许工坐在机房深处，屏幕光照着脸。  
许工：所以它知道自己慢在哪里。

下话：《未闭环事实》
